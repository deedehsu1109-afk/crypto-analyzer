# CryptoAnalyzer 操作手冊

> 本手冊可在系統內開啟：點選視窗右上角「📖 操作手冊」按鈕。
> 本文件會隨系統開發持續更新，最後更新：2026-07-20。

---

## 系統簡介

CryptoAnalyzer 是專為司法調查人員設計的虛擬貨幣鑑識分析工具，用於查詢區塊鏈交易紀錄、建立幣流關聯圖，並產製符合法庭標準的鑑識報告與交易所調閱申請書。支援鏈別：以太坊（ETH）、波場（TRX）、比特幣（BTC）。

系統採「六步驟精靈式」介面，依序為：**歡迎說明 → 使用者身分 → 案件選擇 → 案件資料 → 幣流圖 → 產製報告**。頂部步驟條可隨時點擊切換，但步驟 4 以後需先完成步驟 3（選擇或建立案件）。

---

## 快速上手

1. 開啟系統，閱讀步驟 1 說明後點「我已了解」。
2. 在步驟 2 填寫姓名與識別碼（必填，會納入操作稽核軌跡），點「確認身分」。
3. 在步驟 3 建立新案件，或從清單載入既有案件，點「設為目前作業案件」。
4. 進入步驟 4，於「地址側寫」輸入要查詢的錢包地址，選擇鏈別後按「開始查詢」。
5. 查到結果後可按「加入幣流圖」，切到步驟 5 檢視資金流向。
6. 覺得資料齊全後，回到步驟 6 產製「幣流分析報告」「案件分析報告」，或視需要產製「交易所調閱申請書」。

## 步驟 2：使用者身分確認

**每次開啟系統都需要重新確認**，姓名與識別碼會寫入操作稽核軌跡（`config.json`）。

| 欄位 | 必填 | 說明 |
|---|---|---|
| 姓名 | ✅ | |
| 識別碼 | ✅ | 警員編號／調查員 ID |
| 機關、機關地址、單位、職稱、電話、Email | 否 | 會作為交易所調閱申請書的預設發文機關資訊 |

點「確認身分，進入案件選擇」後即寫入設定並跳至下一步。

### 🔄 檢查更新

頁面標題下方有「🔄 檢查更新」按鈕：

1. 點擊後背景比對本機版本與 GitHub 上的最新版本
2. 已是最新版本 → 顯示「✓ 已是最新版本」
3. 有新版本 → 顯示「🆕 發現新版本」並彈窗列出變更摘要，詢問是否更新
4. 確認更新前系統會自動檢查本機是否有未儲存的修改，若有則中止並提示（不會覆蓋或遺失資料）
5. 更新成功後會彈窗告知，並**自動重新啟動程式**套用變更

系統還會另外偵測「**本程式視窗已經開啟，但本機程式碼在開啟期間又被更新過**」的情況（例如開發階段直接修改了同一份程式檔案）：此時會直接提示「🆕 本機程式碼已變更」並詢問是否重新啟動，**不需要重新從 GitHub 下載**，因為磁碟上的程式碼其實已經是最新的，只是目前這個視窗還在跑啟動當時載入的舊版本。

> 此功能需要本機已安裝 `git` 且能連線存取程式庫，一般情況下不需額外設定。

## 步驟 3：案件選擇

- 從下拉選單選擇既有案件，或按「＋ 新建案件」建立新案件（會開啟完整的案件資料表單）
- 「✎ 編輯選取案件」可修改案件基本資料
- 「🗑 刪除選取案件」會刪除案件本身，但關聯的錢包/Hash 查詢紀錄僅解除關聯、不會被刪除
- 選好案件後點「設為目前作業案件」，或直接按「進入案件資料分析 →」（會自動設定）

之後所有查詢、幣流圖、報告都會關聯到「目前作業案件」，畫面上方會持續顯示案件編號與名稱。

### 案件資料表單分頁

「＋ 新建案件」／「✎ 編輯選取案件」開啟的表單包含三個分頁：

1. **案件基本資料**：案件編號（自動產生）、名稱、類型、狀態、承辦人、筆錄內容（可從文件匯入）等
2. **涉案錢包 / 帳戶**：登記已知的加密錢包地址與一般金融帳戶（靜態資訊，如地址、持有人角色、標記說明）
3. **被害人陳述交易紀錄**：記錄實際的交易往來明細（見下方說明）

### 被害人陳述交易紀錄

案件資料多半來自被害人的口述陳述，不是原生帳本或交易所提供的精確明細——常常說不出精確的交易時間，交易方式也可能混雜銀行轉帳、透過交易所、私人幣商現金交易等多種型態。這個分頁的欄位設計就是為了容納這種不確定性：

- **交易方式**：銀行轉帳／場外交易(OTC)／場內交易(交易所)／透過交易所轉帳／私人幣商現金交易／其他／不明
- **時間精確度**：精確時間／大約時間／僅知先後順序／不確定——選定後，日期/時間欄位可以留白，改用「概略時間描述」填寫如「約下午」「詐騙初期第二筆」「三天內」等口語描述
- **銀行資訊**與**區塊鏈資訊**兩組欄位同時存在、都可以留白——因為同一筆被害人陳述常常同時涉及兩種型態（例如：轉帳到幣商的銀行帳戶，幣商再把虛擬貨幣轉到被害人錢包），不需要勉強歸類成單一類型

新增/編輯記錄時，只需要填寫被害人實際能提供的資訊，其餘留白即可，不強求填滿每一欄。

> **舊版使用者請注意**：本分頁整合了舊版分開的「一般帳戶交易」與「區塊鏈交易」兩個分頁，既有資料已自動遷移過來，不會遺失；只是原本兩張表格合併成一張，並新增了「交易方式」與「時間精確度」欄位。

## 步驟 4：案件資料

進入後左側是「案件資料清單」側欄（可捲動），列出此案件已登記的涉案地址與查詢紀錄，**點擊任一筆會自動填入查詢框並跳到地址側寫分頁查詢**。右側是五個功能分頁：

### 4-A 地址側寫

1. 選擇「一般查詢」（不寫入資料庫）或「專案查詢」（存入資料庫並關聯目前案件）
2. 選擇鏈別（ETH／TRX／BTC），輸入錢包地址（系統會自動判斷地址格式並切換鏈別；若貼上的是交易 Hash 會自動轉去 Hash 分析）
3. 按「開始查詢」，稍候即顯示錢包摘要（17 項統計數據）與授權對象清單
4. 下方「原始交易」「Token 轉帳」分頁可依欄位排序、依地址/Hash/金額/時間搜尋篩選，也可勾選「過濾釣魚交易」隱藏極小額的騷擾交易
5. 需要限定時間範圍時展開「⏱ 時間篩選」：只填起始時間 = 以該時間為中心抓前後各 N 筆；起訖都填 = 抓區間內全部交易
6. 「加入幣流圖」把目前查詢的地址加入幣流圖（僅節點，不含交易邊）
7. 「Excel」「CSV」可將查詢結果匯出成檔案

交易列表右鍵選單可直接「加入發送方/接收方至涉案地址」或「此交易加入幣流圖」，不需重新查詢。

### 4-B Hash 分析

輸入交易 Hash、選鏈別、按「查詢」即顯示該筆交易完整詳情（含 Token 轉帳明細）。「加入幣流圖」會建立發送方→接收方的交易邊。

### 4-C 涉案錢包 / 帳戶

管理此案件所有已知的加密錢包地址與一般金融帳戶：

- 「＋ 新增」手動輸入；「✎ 編輯」修改；「✕ 刪除」批次刪除
- 「📂 從文件提取」可多選 PDF/DOCX/XLSX/ODT/TXT 筆錄文件，系統自動辨識並批次匯入地址與銀行帳號（同時會標示可能的持有人角色與所屬機構）
- 儲存地址前系統會自動檢查同案件內是否重複
- 「⇒ 加入幣流圖」把選取的加密錢包地址加入幣流圖

### 4-D 查詢歷史

顯示此案件所有已存檔的錢包分析與 Hash 查詢紀錄，可搜尋、重新整理、刪除選取列。

### 4-E 網站溯源

輸入可疑網站網域，系統會平行查詢 7 個憑證透明度來源與 3 個被動 DNS 來源，找出繞過 Cloudflare/CDN 保護、可能是真實主機的 IP：

- 結果表格顯示：**IP 位址、ASN 資訊、IP 業者（ISP）、信心度**
- IP 業者欄位會顯示該 IP 實際所屬的機房/雲端業者名稱與國家（例：`Amazon.com, Inc. (AWS) · United States`），比單純的 ASN 代碼更容易直接判讀
- 也會列出子網域解析結果
- 選取任一筆 IP，按「加入涉案地址」即可寫入此案件的涉案地址清單（機構欄位會自動帶入查到的 IP 業者名稱）

## 步驟 5：幣流關聯圖

> 詳細的按鈕功能對照表請見下方「幣流圖工具列詳解」。這裡先說明整體操作邏輯。

畫面分為：上方主工具列、圖表區（含第二排的畫布導覽列）、下方狀態列。

### 四種視圖

| 視圖 | 適合用途 |
|---|---|
| 地址關係圖 | 看整體資金拓撲，節點=地址、邊=聚合交易；**唯一支援拖曳節點與連接線折點的視圖** |
| 幣流流水圖 | 依時間排序逐筆呈現，還原犯罪路徑 |
| 時序泳道圖 | 找出多名被害人資金匯聚的時間點 |
| 司法金流圖 | 階層式排版，每筆交易獨立顯示時間+金額，適合當作法庭簡報主圖 |

### 地址關係圖操作

- **左鍵拖曳節點**：調整節點位置（按住 Shift 可複選多個節點一起拖）
- **右鍵點節點**：查詢此地址／展開節點／標記角色／加入涉案錢包／複製地址
- **連接線是直線**，如果多條線重疊不好看，可以**右鍵點連接線 →「➕ 在此新增折點」**，新增一個可拖曳的轉折點來改變線的走向；折點顯示為橘色菱形，左鍵拖曳即可調整位置；右鍵點折點可「🗑 刪除此折點」
- 節點位置與折點位置都可以用「更新案件圖」存進案件資料，下次開啟案件會自動還原

### 幣流圖工具列詳解

**主工具列（最上排）：**

| 按鈕 | 功能 | 使用時機 |
|---|---|---|
| 視圖切換 | 切換四種視圖 | 依分析需求切換呈現方式 |
| 重新佈局 | 清空目前座標與折點，重新自動排版 | 手動排太亂想重來時 |
| 尋找路徑 | 輸入起訖地址前幾碼，列出所有金流路徑 | 想確認資金是否間接流向某地址 |
| 標記節點 | 為選取節點加自訂標籤 | 想標註「幣商甲」等好辨識的名稱 |
| 更新案件圖 | 存入案件資料庫（覆蓋舊快照） | **手動調整後務必按這個才會存檔** |

| 清除圖 | 清空目前畫面（不影響已存快照） | 想從空白重新開始 |

| 💾 存檔／📂 讀檔 | 匯出/匯入完整圖表為 .json 檔案 | 需要備份或跨案件搬移圖表時 |

**畫布導覽列（圖表正上方第二排）：**

| 按鈕 | 功能 |
|---|---|
| ⌂ 還原視圖 | 回到初始縮放範圍 |
| ← 退回／→ 前進 | 切換上一步/下一步的檢視範圍 |
| ✥ 平移移動 | 啟用後可拖曳整個畫面（會暫時取代拖曳節點的功能，用完記得關閉） |
| ⊕ 框選縮放 | 框選一個範圍放大檢視 |
| 💾 儲存圖片 | 另存截圖（與主工具列「匯出圖片」功能重疊） |

## 步驟 6：產製報告

上方顯示目前案件與輸出目錄（可點「瀏覽」選擇）。

### 幣流分析報告

依案件的幣流資料自動產製「幣流分析專家意見書」Word 文件（壹～玖章節：案件背景、資料來源、幣流事實、供述比對、斷點分析、錢包剖繪、結論建議、附錄）。按「▶ 產製幣流分析報告」，日誌區會即時顯示產製進度。

### 案件分析報告

依案件基本資料產製「虛擬貨幣詐欺案件分析範本」Word 文件（九大章節）。按「▶ 產製案件分析報告」。

### 交易所調閱申請書

頁面最下方「📨 交易所調閱申請書」卡片，點「開啟申請書填表」進入填表視窗（依 OKX 官方範本格式，支援輸出 DOCX / ODT / PDF）：

- **交易所名稱**欄位可直接輸入文字，打字時會自動彈出符合的交易所清單（目前收錄 50 家交易所的受文資訊，含執法聯絡 email），選取後自動帶入對應的受文 Email；清單依字母排序，可用滑鼠滾輪瀏覽
- 部分交易所僅提供線上申請入口網站、沒有公開直接 email，此時 Email 欄位會留空，需另外查詢（詳見 `D:\claude\交易所調閱資訊彙整表.md` 或 `.csv`，內含 48 家交易所的完整調閱流程說明）
- 其餘欄位（發文機關、案件性質、案情摘要、涉案錢包、調閱期間、附件等）依序填寫後按「產製申請書」即可輸出符合官方格式的文件

## 常見問題 FAQ

**Q：查詢一直轉圈圈沒有結果？**
A：檢查右上角「⚙ 系統設定」是否已填入 Etherscan API Key（TRX/BTC 不需要 Key）。也可能是該地址交易量過大，系統會在超過 1000 筆時跳出警告視窗，可選擇改用時間篩選縮小範圍。

**Q：步驟 4/5/6 按不進去？**
A：需要先在步驟 3 選擇或建立一個案件，系統會自動導回步驟 3 並提示。

**Q：幣流圖手動調整的位置怎麼不見了？**
A：拖曳節點/折點只存在畫面記憶體中，離開頁面前要按「更新案件圖」才會真正寫入資料庫。

**Q：交易所調閱申請書的受文 Email 是空的？**
A：代表該交易所目前只提供線上申請入口網站、沒有公開直接 email，請至 `D:\claude\交易所調閱資訊彙整表.md` 查詢對應的入口網址與申請流程。

**Q：檢查更新一直失敗？**
A：確認本機已安裝 git 且能連線到 GitHub；若本機有尚未儲存的修改，系統會主動中止更新以避免覆蓋資料，需先自行處理後再試。

**Q：明明剛推送了新的修正到 GitHub，程式視窗卻一直開著，按「檢查更新」卻說已是最新版本？**
A：這是正常現象。「檢查更新」比對的是「本機程式碼」與「GitHub 遠端」的差異；如果新修正是直接在本機這份程式碼上完成並推送（開發階段常見情況），推送完成後本機與 GitHub 其實已經一致，自然不會顯示有更新可下載。但目前這個「已經開著的視窗」仍是啟動當時載入的舊版本——這種情況系統會改顯示「🆕 本機程式碼已變更」，直接提示重新啟動即可（不需要等 GitHub 那邊的比對）。若兩種提示都沒出現，可以直接手動關閉視窗、重新開啟程式即可套用最新版本。
